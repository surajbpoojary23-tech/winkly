# Winkly – Telegram Dating Bot

A lightweight matchmaking bot that works with location‑based matching and a **match‑relay** pattern to keep conversations private.

## Features
- Users share their location (via Telegram location button).
- Bot finds nearby users (5 km radius) using Redis GEO.
- Creates a match with a free‑message quota of 10 messages per user.
- Messages are forwarded through the bot (`bot.copy_message`) so users never exchange direct IDs.
- Simple `/stop` command to end a match.
- Deployable on any container platform (Render, Fly.io, Render.com) using Docker.

## Quick Start (local)
```bash
# Clone / create directory
mkdir -p ~/winkly_bot && cd ~/winkly_bot
# (files already placed by the assistant)
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
# Create .env with your credentials
cp .env.example .env
# Edit .env and set:
#   BOT_TOKEN=YOUR_TELEGRAM_BOT_TOKEN
#   REDIS_URL=YOUR_UPSTASH_REDIS_URL (e.g. redis://...)
#   ALLOW_LONG_POLLING=true  # local-only; do not set this on Render web services
python bot.py
```
The bot will start polling. Open Telegram, send `/start`, share your location and tap *❤️  Find Matches*.

## Deployment on Render (free tier)
1. Create a new **Web Service** on Render linked to the Git repo (or push the local folder).
2. Set **Build Command**: `pip install -r requirements.txt`
3. Set **Start Command**: `python bot.py`
4. Add **Environment Variables**:
   - `BOT_TOKEN` – the token you received from @BotFather.
   - `REDIS_URL` – your Upstash Redis endpoint (e.g. `rediss://...`).
   - `PORT` – port to bind to (default: 8080, required for webhook mode)
   - `WEBHOOK_URL` – public base URL (default: `https://winkly-kmsz.onrender.com`)
   - `TELEGRAM_WEBHOOK_PATH` – Telegram webhook path (default: `/telegram/webhook`)
   - `TELEGRAM_WEBHOOK_SECRET` – optional random 32+ character Telegram webhook secret; if omitted, the bot derives one from `BOT_TOKEN`
   - `RAZORPAY_KEY_ID`, `RAZORPAY_KEY_SECRET`, `RAZORPAY_WEBHOOK_SECRET`
   - Do not set `ALLOW_LONG_POLLING=true` on Render web services.
5. Deploy. Render will build the Docker image and start the container.

### ⚠️ Important: Fix TelegramConflictError
The error "Conflict: terminated by other getUpdates request" occurs when multiple bot instances run simultaneously. To fix this:

**Option 1: Use Webhook Mode (Recommended)**
- Set `WEBHOOK_URL` to your Render service base URL (`https://winkly-kmsz.onrender.com`) or use the built-in default
- Set `TELEGRAM_WEBHOOK_SECRET` or let the bot derive it from `BOT_TOKEN`
- This eliminates polling conflicts and is more reliable for production

**Option 2: Use Long-Polling with Single Instance**
- Ensure only one bot instance runs
- Set `skip_updates=True` to skip pending updates on restart

**Option 3: Use Render's Health Check**
- Add a health check endpoint to verify bot is running
- Use Render's auto-scaling to prevent multiple instances

## How It Works (internals)
- **Redis GEO** (`dating:users`) stores each user’s longitude/latitude.
- **Match ID** is generated with an atomic `INCR` (`dating:match:id`).
- **Match hash** (`dating:match:{id}`) holds `user_a`, `user_b`, `msgs_a`, `msgs_b`.
- **User‑to‑match key** (`dating:user:{uid}:match`) links a user to their current match.
- **Free‑message limit** is enforced by decrementing `msgs_a`/`msgs_b` on every forward.
- When the limit hits zero, the bot informs the user and suggests an upgrade (payment integration can be added later).

## Extending
- **Payment**: add a `/upgrade` flow that calls Razorpay/PayPal, then reset the counters via `HSET`.
- **Radius/config**: expose a `/radius <km>` command to let users adjust search distance.
- **Filtering**: store extra preference fields in another Redis hash and filter candidates before matching.

## Security & Pitfalls
- Keep the bot token secret – never commit it.
- Upstash Redis connections are TLS‑encrypted by default.
- Users who block the bot will cause `TelegramForbiddenError`; the bot clears the match for that user.
- Rate‑limit each user with a token bucket if you expect high traffic.
